# Programming Language 기말 구술 평가 대비 정리









# 1 infinite

1. **Interpreter란 무엇인가요?**

   - **답변:** 인터프리터는 프로그램을 입력으로 받아 출력으로 변환하는 소프트웨어입니다. 프로그램을 데이터로 취급하며, 귀납적으로 정의된 데이터를 처리할 수 있습니다.

2. **Tiny Language에서 무한한 표현식을 작성할 수 있는 이유는 무엇인가요?**

   - **답변:** Tiny Language는 유한한 재료(Num, Exp + Exp)로 구성되어 있지만, 이 재료들을 조합하여 무한한 수의 표현식을 생성할 수 있습니다. 이는 재귀적인 정의 덕분입니다.

3. **Inductive Definition에서 집합 S의 원소는 어떻게 결정되나요?**

   - **답변:** 집합 S는 자연수 n이 0이거나 n - 3이 S에 속할 때 n이 S에 속하는 방식으로 정의됩니다. 예를 들어, 0, 3, 6, 9 등이 S에 속합니다.

4. **정수 리스트의 길이를 계산하는 함수는 어떻게 구현되나요?**

   - **답변:** 리스트의 길이를 계산하는 함수는 리스트가 비어 있는지 확인하고, 비어 있지 않다면 리스트의 머리(head)를 제외한 꼬리(tail)의 길이를 재귀적으로 계산하여 구현됩니다.

   - 예를 들어,

   - ```scheme
     (define list-length (lambda (lst) (if (null? lst) 0 (+ 1 (list-length (cdr lst))))))
     ```

   - 와 같이 작성할 수 있습니다.

5. **nth-element 함수는 무엇을 하나요?**

   - **답변:** nth-element 함수는 리스트에서 n번째 요소를 반환합니다. 예를 들어, `(nth-element '(a b c) 1)`은 `b`를 반환합니다. 이 함수는 리스트의 첫 번째 요소를 반환하거나, n이 0이 될 때까지 리스트의 꼬리를 재귀적으로 탐색합니다.











# 2 rep

1. **Lambda Calculus란 무엇인가요?**

   - **답변:** 람다 계산법은 튜링 기계만큼 강력한 간단한 계산 모델로, 함수 정의와 적용을 통해 계산을 수행합니다.

2. **람다 계산법에서 자연수를 어떻게 표현하나요?**

   - **답변:** 람다 계산법을 사용하여 자연수를 인코딩할 수 있으며, 이를 Church 수라고 합니다.

   - 예를 들어,

     0: (lambda (f) (lambda (x) x))

     1: (lambda (f) (lambda (x) (f x)))

     2: (lambda (f) (lambda (x) (f (f x))))

3. **occurs-free? 함수는 무엇을 하나요?**

   - **답변:** occurs-free? 함수는 주어진 변수(var)가 표현식(exp) 내에서 자유롭게 발생하는지 여부를 확인합니다.

4. **추상 구문 트리(AST)란 무엇인가요?**

   - **답변:** 추상 구문 트리는 프로그램의 구문 구조를 트리 형태로 표현한 것으로, 각 노드는 프로그램의 구성 요소를 나타냅니다.

5. **패턴 매칭이란 무엇인가요?**

   - **답변:** 패턴 매칭은 주어진 패턴에 따라 데이터를 검사하고, 일치하는 경우 해당 데이터를 처리하는 방법입니다.

6. **lc-exp 표현식을 처리하기 위한 연산자는 무엇인가요?**

   - **답변:** lc-exp 표현식을 처리하기 위한 연산자로는 var-exp?, lambda-exp?, app-exp? 등이 있으며, 추출자로는 var-exp->var, lambda-exp->bound-var, app-exp->rator 등이 있습니다.

7. **람다 계산법 인터프리터는 어떻게 동작하나요?**

   - **답변:** 람다 계산법 인터프리터는 람다 계산법 표현식을 입력으로 받아 이를 평가하여 결과를 출력합니다.

8. **Church 수란 무엇인가요?**

   - **답변:** Church 수는 람다 계산법을 사용하여 자연수를 표현한 것으로, 각 자연수를 람다 표현식으로 인코딩한 것입니다.

9. **추상 구문 트리의 예를 들어보세요.**

   - **답변:** 예를 들어, `((λx.x) 5)`는 추상 구문 트리로 표현하면 `app-exp` 노드 아래에 `lambda-exp`와 `const-exp` 노드가 위치합니다.

10. **패턴 매칭 없이 표현식을 처리하는 방법은 무엇인가요?**

    - **답변:** 패턴 매칭 없이 표현식을 처리하려면 조건문을 사용하여 각 표현식의 유형을 검사하고, 해당 유형에 맞는 처리를 수행합니다.









# 3 env

1. **Interpreter란 무엇인가요?**

   - **답변:** 인터프리터는 프로그램을 입력으로 받아 추상 구문 트리(AST)로 변환하고, 이를 평가하여 출력으로 변환하는 소프트웨어입니다.

2. **Front End는 무엇으로 구성되나요?**

   - **답변:** Front End는 Lexer와 Parser로 구성됩니다.

3. **구체 구문(Concrete Syntax)과 추상 구문(Abstract Syntax)의 차이는 무엇인가요?**

   - **답변:** 구체 구문은 프로그램의 실제 문법 구조를 나타내고, 추상 구문은 이를 간단하게 표현한 구조입니다.

4. **상수 표현식(const-exp)을 평가하는 방법은 무엇인가요?**

   - **답변:** 상수 표현식은 숫자를 그대로 반환합니다. 예를 들어, `const-exp 5`는 `5`를 반환합니다.

5. **변수 표현식(var-exp)을 평가하는 방법은 무엇인가요?**

   - **답변:** 변수 표현식은 환경에서 해당 변수의 값을 찾아 반환합니다.

6. **환경(Environment)이란 무엇인가요?**

   - **답변:** 환경은 변수와 값을 매핑하는 함수로, 인터프리터가 변수를 평가할 때 사용됩니다.

7. **빈 환경(empty-env)은 어떻게 정의되나요?**

   - **답변:** 빈 환경은 아무런 변수도 포함하지 않는 환경으로,

     ```scheme
     (define empty-env (lambda () (list 'empty-env)))
     ```

     와 같이 정의됩니다.

8. **확장된 환경(extend-env)은 어떻게 구현되나요?**

   - **답변:** 확장된 환경은 기존 환경에 새로운 변수와 값을 추가한 환경으로,

   - ```scheme
     (define extend-env
       (lambda (var val env) (list 'extend-env var val env)))
     ```

   -  와 같이 구현됩니다.

9. **환경에서 변수를 찾는 함수(apply-env)는 어떻게 동작하나요?**

   - **답변:** `apply-env` 함수는 환경에서 변수를 찾아 해당 값을 반환합니다. 만약 변수가 환경에 없으면 오류를 보고합니다.

10. **let 표현식은 무엇인가요?**

    - **답변:** let 표현식은 새로운 변수를 정의하고, 그 변수를 사용하여 표현식을 평가하는 구문입니다. 예를 들어, `let x = 5 in x + 3`은 `8`을 반환합니다.









# 4 spec-impl

### 질문과 답변

1. **let 표현식이란 무엇인가요?**

   - **답변:** let 표현식은 새로운 변수를 정의하고, 그 변수를 사용하여 표현식을 평가하는 구문입니다. 예를 들어, `let x = 5 in x + 3`은 `8`을 반환합니다.

2. **let 표현식의 평가 방법은 무엇인가요?**

   - **답변:** let 표현식은 먼저 변수에 할당된 값을 평가한 후, 그 값을 사용하여 본문(body)을 평가합니다. 예를 들어, `let x = 5 in x + 3`은 `x`에 `5`를 할당한 후 `x + 3`을 평가하여 `8`을 반환합니다.

3. **zero? 표현식이란 무엇인가요?**

   - **답변:** zero? 표현식은 주어진 표현식의 값이 0인지 여부를 확인하는 구문입니다. 예를 들어, `zero?(5)`는 `#f`를 반환하고, `zero?(0)`는 `#t`를 반환합니다.

4. **zero? 표현식의 평가 방법은 무엇인가요?**

   - **답변:** zero? 표현식은 먼저 내부 표현식을 평가한 후, 그 값이 0인지 확인합니다. 값이 0이면 `#t`를, 그렇지 않으면 `#f`를 반환합니다.

5. **조건문이란 무엇인가요?**

   - **답변:** 조건문은 주어진 조건에 따라 다른 표현식을 평가하는 구문입니다. 예를 들어, `if x then y else z`는 `x`가 참이면 `y`를, 거짓이면 `z`를 평가합니다.

6. **조건문의 평가 방법은 무엇인가요?**

   - **답변:** 조건문은 먼저 조건 표현식을 평가한 후, 그 값이 참이면 then 절을, 거짓이면 else 절을 평가합니다. 예를 들어, `if 1 then 2 else 3`은 `2`를 반환합니다.

7. **let 표현식의 명세와 구현 예제를 들어보세요.**

   - **답변:**

   - spec:

   - ```scheme
     (value-of (let-exp var exp body) ρ) = (value-of body [var = val]ρ)
     ```

   - 구현:

   - ```scheme
     (define value-of
       (lambda (exp env)
         (cases expression exp
                (let-exp (var exp body)
                         (let ((val (value-of exp env)))
                           (value-of body (extend-env var val env)))))))
     ```

   -

8. **zero? 표현식의 명세와 구현 예제를 들어보세요.**

   - **답변:**

   - spec:

   - ```scheme
     (value-of (zero?-exp exp) ρ) = (bool-val #t)
     	if (expval->num val) = 0 (bool-val #f)
     	if (expval->num val) ≠ 0
     ```

   -  구현:

   - ```scheme
     (define value-of
       (lambda (exp env)
         (cases expression exp
                (zero?-exp (exp)
                           (let ((val (value-of exp env)))
                             (let ((num (expval->num val)))
                               (if (zero? num) (bool-val #t) (bool-val #f)))))))
     ```

   -

9. **조건문의 명세와 구현 예제를 들어보세요.**

   - **답변:** 명세:

   - ```scheme
     (value-of (if-exp exp1 exp2 exp3) ρ)
     = (value-of exp2 ρ)
     	if (expval->bool val1) = #t (value-of exp3 ρ)
       if (expval->bool val1) = #f
     ```

   -  구현:

   - ```scheme
     (define value-of
       (lambda (exp env)
         (cases expression exp
                (if-exp (exp1 exp2 exp3)
                        (let ((val1 (value-of exp1 env)))
                          (if (expval->bool val1)
                              (value-of exp2 env)
                              (value-of exp3 env)))))))
     ```

   -

10. **구체 구문과 추상 구문의 차이는 무엇인가요?**

    - **답변:** 구체 구문은 프로그램의 실제 문법 구조를 나타내고, 추상 구문은 이를 간단하게 표현한 구조입니다. 구체 구문은 프로그램의 텍스트 형태를, 추상 구문은 프로그램의 논리적 구조를 나타냅니다.









# 5 proc

1. **절차 표현식(proc-exp)이란 무엇인가요?**

   - **답변:** 절차 표현식은 함수 정의를 나타내며, 형식 매개변수와 본문으로 구성됩니다. 예를 들어, `proc (x) x + 1`은 `x`를 매개변수로 받아 `x + 1`을 반환하는 절차입니다.

2. **호출 표현식(call-exp)이란 무엇인가요?**

   - **답변:** 호출 표현식은 절차를 호출하는 구문으로, 연산자와 피연산자로 구성됩니다. 예를 들어, `(f 1)`은 절차 `f`를 호출하여 인수 `1`을 전달합니다.

3. **Python에서 함수 정의와 호출 예제를 들어보세요.**

   - **답변:** Python에서 함수는 `def` 키워드를 사용하여 정의합니다. 예를 들어, `def inc(x): return x + 1`은 `x`를 1 증가시키는 함수입니다. 이를 호출하려면 `inc(5)`와 같이 사용합니다.

4. **절차의 명세는 어떻게 작성되나요?**

   - **답변:** 절차의 명세는 절차 표현식을 평가하여 절차 값을 반환하는 방식으로 작성됩니다. 예를 들어,

   - ```scheme
     (value-of (proc-exp var body) ρ) = (proc-val (procedure var body ρ))
     ```

   - 와 같이 작성됩니다.

5. **절차 예제에서 환경(ρ)의 역할은 무엇인가요?**

   - **답변:** 환경(ρ)은 변수와 값을 매핑하는 역할을 하며, 절차 평가 과정에서 변수의 값을 찾는 데 사용됩니다. 예제에서는 환경이 단계별로 업데이트되며, 각 단계에서 변수의 값을 추적합니다.

6. **클로저란 무엇인가요?**

   - **답변:** 클로저는 절차와 그 절차가 정의된 환경을 포함하는 데이터 구조입니다. 클로저는 절차를 평가하는 데 필요한 모든 정보를 포함합니다.

7. **절차 값을 생성하는 방법은 무엇인가요?**

   - **답변:** 절차 값은 절차 표현식을 평가하여 생성됩니다. 예를 들어, `(proc-val (procedure var body env))`와 같이 생성됩니다.

8. **절차 호출을 평가하는 방법은 무엇인가요?**

   - **답변:** 절차 호출은 먼저 연산자와 피연산자를 평가한 후, 절차를 호출하여 결과를 반환합니다. 예를 들어,

   - ```scheme
     (value-of (call-exp rator rand) ρ) = (apply-procedure proc arg)
     ```

   - 와 같이 평가됩니다.

9. **Python에서 함수의 메모리 주소를 출력하는 예제를 들어보세요.**

   - **답변:** Python에서 함수의 메모리 주소는 함수 객체를 출력하여 확인할 수 있습니다. 예를 들어, `def inc(x): return x + 1`을 정의한 후 `print(inc)`를 실행하면 함수의 메모리 주소가 출력됩니다.

10. **절차 예제에서 let 표현식의 역할은 무엇인가요?**

    - **답변:** let 표현식은 새로운 변수를 정의하고, 그 변수를 사용하여 표현식을 평가하는 구문입니다. 예제에서는 여러 개의 let 표현식을 사용하여 변수와 절차를 정의하고, 이를 호출하여 결과를 계산합니다.



# 6 recursion

1. **재귀 절차란 무엇인가요?**

   - **답변:** 재귀 절차는 자기 자신을 호출하는 함수로, 주어진 조건이 만족될 때까지 반복적으로 호출됩니다. 예를 들어, `double` 함수는 `x`가 0이 될 때까지 `x`를 1씩 감소시키며 2를 곱합니다.

2. **재귀 절차의 평가 방법은 무엇인가요?**

   - **답변:** 재귀 절차는 먼저 기본 조건(base case)을 확인하고, 기본 조건이 만족되지 않으면 재귀적으로 자신을 호출하여 결과를 계산합니다. 예를 들어, `double` 함수는 `x`가 0이 아니면 `double(x-1)`을 호출하여 결과를 계산합니다.

3. **letrec 표현식이란 무엇인가요?**

   - **답변:** letrec 표현식은 재귀 절차를 정의하는 구문으로, 절차 이름과 본문을 포함합니다. 예를 들어, `letrec double = proc (x) ... in ...`은 `double`이라는 재귀 절차를 정의합니다.

4. **letrec 표현식의 평가 방법은 무엇인가요?**

   - **답변:** letrec 표현식은 먼저 재귀 절차를 정의한 후, 그 절차를 사용하여 본문을 평가합니다. 예를 들어, `letrec double = proc (x) ... in (double 6)`은 `double` 절차를 정의한 후 `double(6)`을 평가합니다.

5. **apply-env 함수는 무엇을 하나요?**

   - **답변:** apply-env 함수는 환경에서 변수를 찾아 해당 값을 반환합니다. 재귀 환경에서는 재귀적으로 환경을 확장하고, 변수를 찾을 때까지 이전 환경을 탐색합니다.

6. **재귀 절차 평가 시 발생할 수 있는 문제는 무엇인가요?**

   - **답변:** 재귀 절차 평가 시 발생할 수 있는 문제는 절차가 정의된 환경을 올바르게 참조하지 못하는 경우입니다. 이를 해결하기 위해 재귀 환경을 올바르게 확장하고 적용해야 합니다.

7. **재귀 절차의 예제를 들어보세요.**

   - **답변:** 예를 들어,

   - ```scheme
     letrec double = proc (x)
     	if zero?(x) then 0
       else -((double -(x,1)), -2) in (double 6)
     ```

   - 은 `double`이라는 재귀 절차를 정의하고, `double(6)`을 호출하여 결과를 계산합니다.

8. **재귀 환경을 확장하는 방법은 무엇인가요?**

   - **답변:** 재귀 환경을 확장하려면 `extend-env-rec` 함수를 사용하여 절차 이름과 본문을 환경에 추가합니다. 예를 들어,

   - ```scheme
     extend-env-rec double x <<if zero?(x) ... >> ρ₀
     ```

   - 와 같이 확장합니다.

10. **재귀 절차 평가 시 환경의 변화를 설명해 주세요.**

    - **답변:** 재귀 절차 평가 시 환경은 절차 호출마다 확장됩니다. 예를 들어, `double(6)`을 호출하면 `[x=6]ρ₀`로 확장되고, `double(5)`을 호출하면 `[x=5]ρ₀`로 확장됩니다. 이 과정을 반복하여 최종 결과를 계산합니다.





# 7 scoping







1. **렉시컬 스코핑이란 무엇인가요?**

   - **답변:** 렉시컬 스코핑은 변수의 범위가 코드의 구조에 따라 결정되는 방식입니다. 변수는 정의된 블록 내에서만 유효하며, 외부 블록에서는 접근할 수 없습니다.

2. **다이나믹 스코핑이란 무엇인가요?**

   - **답변:** 다이나믹 스코핑은 변수의 범위가 실행 시간에 따라 결정되는 방식입니다. 변수는 호출된 함수의 실행 컨텍스트에 따라 값이 결정됩니다.

3. **let 표현식의 예제를 설명해 주세요.**

   - **답변:** 예를 들어,

   - ```scheme
     let z = 5
       in let x = 3
         in let y = -(x,1)
           in let x = 4
             in -(z, -(x,y))
     ```

   -

   - 는 여러 개의 let 표현식을 중첩하여 변수의 범위를 설명합니다. 각 변수는 해당 블록 내에서만 유효합니다.

4. **절차 예제에서 변수의 값은 어떻게 결정되나요?**

   - **답변:** 절차 예제에서는 절차가 정의된 환경에 따라 변수의 값이 결정됩니다. 예를 들어,

   - ```scheme
     let x = 200
     	in let f = proc (z) -(z,x)
       	in let x = 100
         	in let g = proc (z) -(z,x)
           	in -((f 1),(g 1))
     ```

   - 에서 `f`와 `g`는 각각 다른 `x` 값을 참조합니다.

5. **컨투어 다이어그램이란 무엇인가요?**

   - **답변:** 컨투어 다이어그램은 변수의 범위를 시각적으로 나타낸 다이어그램입니다. 각 블록은 변수의 범위를 나타내며, 중첩된 블록은 변수의 중첩된 범위를 나타냅니다.

6. **정적 스코핑과 동적 스코핑의 차이점은 무엇인가요?**

   - **답변:** 정적 스코핑은 변수의 범위가 코드의 구조에 따라 결정되고, 동적 스코핑은 실행 시간에 따라 결정됩니다. 정적 스코핑에서는 변수의 범위가 고정되어 있지만, 동적 스코핑에서는 호출된 함수의 실행 컨텍스트에 따라 달라집니다.

7. **바인딩 규칙이란 무엇인가요?**

   - **답변:** 바인딩 규칙은 변수에 값을 할당하는 방법을 설명합니다. 특정 시점에서 변수에 어떤 값이 할당되는지를 결정합니다.

8. **렉시컬 스코핑의 예제를 들어보세요.**

   - **답변:** 예를 들어,

     ```scheme
     let x = 200
      	in let f = proc (z) -(z,x)
       	in let x = 100
       		in let g = proc (z) (z x)
       			in (g f)
     ```

   - 에서 `f`는 `x=200`을 참조하고, `g`는 `x=100`을 참조합니다.

9. **동적 스코핑의 예제를 들어보세요.**

   - **답변:** 예를 들어,

     ```scheme
     let a = 3 in let p = proc (x) -(x,a) in let a = 5 in -(a, (p 2))
     ```

     `p`는 호출된 시점의 `a` 값을 참조하여 `-(2,5)`를 계산합니다.

10. **스코핑 규칙이 변수의 값을 결정하는 방법은 무엇인가요?**

    - **답변:** 스코핑 규칙은 변수의 범위와 바인딩을 결정하여 특정 시점에서 변수의 값을 결정합니다. 정적 스코핑에서는 코드 구조에 따라, 동적 스코핑에서는 실행 컨텍스트에 따라 값이 결정됩니다.

































# 8 sugar



### 주요 내용 요약

1. **Multiple Arguments (다중 인수)**

   - 다중 인수를 사용하는 절차를 정의하고 호출하는 예제를 설명합니다.
   - 예제: `let f = proc (x,y) -(x,y) in (f (4, 3))`

2. **Extending the Language (언어 확장)**

   - 구체 구문(Concrete Syntax)과 추상 구문(Abstract Syntax)을 확장하는 방법을 설명합니다.
   - 파서(Parser)와 평가기(Evaluator)를 확장하는 방법을 설명합니다.

3. **Extending the Evaluator (평가기 확장)**

   - 다중 인수를 처리하기 위해 평가기를 확장하는 방법을 설명합니다.
   - 예제: `(value-of (proc-exp (args x y) body) ρ) = (proc-val (procedure (x y) body ρ))`

4. **Transformation (변환)**

   - 다중 인수를 단일 인수로 변환하는 방법을 설명합니다.

   - 예제:

   - ```scheme
     let f = proc (x,y) -(x,y)
     	in (f (4, 3))
     ```

   - 를

   - ```scheme
     let f = proc (x) proc (y) -(x,y)
     	in ((f 4) 3)
     ```

   - 로 변환

5. **Syntactic Sugar (문법 설탕)**

   - 문법 설탕이란 무엇인지 설명합니다.

   - 문법 설탕은 언어의 의미적 표현력을 변경하지 않으면서 사용하기 쉽게 만드는 구문적 변형입니다.

   - 예제:

   - ```scheme
     let f = proc (x,y) -(x,y)
     	in (f (4, 3))
     ```

   - 와

   - ```scheme
     let f = proc (x) proc (y) -(x,y)
     	in ((f 4) 3)
     ```



### 질문과 답변

1. **문법 설탕(Syntactic Sugar)이란 무엇인가요?**

   - **답변:** 문법 설탕은 프로그래밍 언어를 더 쉽게 사용하기 위해 구문을 변형하는 것으로, 언어의 의미적 표현력을 변경하지 않습니다. 예를 들어, 다중 인수를 단일 인수로 변환하는 것이 문법 설탕의 한 예입니다.

2. **다중 인수를 사용하는 절차를 정의하는 방법은 무엇인가요?**

   - **답변:** 다중 인수를 사용하는 절차는 `proc (x,y) -(x,y)`와 같이 정의할 수 있습니다. 이는 두 개의 인수를 받아 그 차이를 반환하는 절차입니다.

3. **다중 인수를 단일 인수로 변환하는 방법은 무엇인가요?**

   - **답변:** 다중 인수를 단일 인수로 변환하려면 각 인수를 개별적으로 처리하는 절차로 변환합니다. 예를 들어, `proc (x,y) -(x,y)`를 `proc (x) proc (y) -(x,y)`로 변환할 수 있습니다.

4. **언어를 확장하는 방법은 무엇인가요?**

   - **답변:** 언어를 확장하려면 구체 구문과 추상 구문을 확장하고, 파서와 평가기를 수정해야 합니다. 이를 통해 새로운 구문을 처리할 수 있습니다.

5. **평가기를 확장하는 방법은 무엇인가요?**

   - **답변:** 평가기를 확장하려면 새로운 구문을 처리할 수 있도록 `value-of` 함수를 수정해야 합니다. 예를 들어, 다중 인수를 처리하기 위해 `(value-of (proc-exp (args x y) body) ρ)`와 같이 수정할 수 있습니다.

6. **문법 설탕의 예를 들어보세요.**

   - **답변:** 예를 들어, `let f = proc (x,y) -(x,y) in (f (4, 3))`는 문법 설탕을 사용하여

     ```scheme
     let f = proc (x) proc (y) -(x,y) in ((f 4) 3)
     ```

     로 변환할 수 있습니다. 이는 동일한 의미를 가지지만, 구문적으로 더 간단합니다.

7. **문법 설탕이 언어의 의미적 표현력에 미치는 영향은 무엇인가요?**

   - **답변:** 문법 설탕은 언어의 의미적 표현력에 영향을 미치지 않습니다. 단지 구문을 더 간단하고 사용하기 쉽게 만드는 역할을 합니다.

8. **다중 인수를 처리하는 평가기의 예를 들어보세요.**

   - **답변:** 예를 들어, `(value-of (proc-exp (args x y) body) ρ)`는 `(proc-val (procedure (x y) body ρ))`로 평가할 수 있습니다. 이는 두 개의 인수를 받아 본문을 평가하는 절차를 반환합니다.

9. **구체 구문과 추상 구문의 차이는 무엇인가요?**

   - **답변:** 구체 구문은 프로그램의 실제 문법 구조를 나타내고, 추상 구문은 이를 간단하게 표현한 구조입니다. 구체 구문은 프로그램의 텍스트 형태를, 추상 구문은 프로그램의 논리적 구조를 나타냅니다.

10. **파서를 확장하는 방법은 무엇인가요?**

    - **답변:** 파서를 확장하려면 새로운 구문을 인식하고 처리할 수 있도록 파서의 규칙을 수정해야 합니다. 예를 들어, 다중 인수를 처리하기 위해 파서에 `(x,y)`와 같은 구문을 추가할 수 있습니다.





# 9 state

### 주요 내용 요약

1. **State (상태)**

   - 상태를 관리하는 방법과 관련된 예제를 설명합니다.

   - 예제:

     ```scheme
     let counter = 0
     	in let f = proc (x) (let counter = counter + 1 in x)
     		in let a = (f (f 1)) in counter
     ```

2. **Store (메모리)**

   - 상태를 저장하는 메모리 구조를 설명합니다.
   - 환경(Environment)과 저장소(Store)의 관계를 설명합니다.

3. **Store-Passing Specification (저장소 전달 명세)**

   - 저장소를 전달하면서 표현식을 평가하는 방법을 설명합니다.
   - 예제: `(value-of (const-exp n) ρ σ₀) = (n, σ₀)`

4. **Store-Manipulating Operations (저장소 조작 연산)**

   - 저장소를 조작하는 연산을 설명합니다.
   - 새로운 참조 생성(newref), 참조 해제(deref), 참조 설정(setref) 등의 연산을 설명합니다.

5. **Explicit Reference (명시적 참조)**

   - 명시적 참조를 사용하여 상태를 관리하는 방법을 설명합니다.

     ```scheme
     let counter = newref(0)
     	in let f = proc (x) setref(counter, deref(counter) + 1)
       	in let a = (f (f 1))
         	in deref(counter)
     ```

6. **Implicit Reference (암시적 참조)**

   - 암시적 참조를 사용하여 상태를 관리하는 방법을 설명합니다.

     ```scheme
     let counter = 0
     	in let f = proc (x) set counter = counter + 1
       	in let a = (f (f 1))
         	in counter
     ```

7. **L-Value와 R-Value**

   - L-Value와 R-Value의 개념을 설명합니다.
   - 변수의 값과 참조를 구분하여 설명합니다.

### 질문과 답변

1. **상태(State)란 무엇인가요?**

   - **답변:** 상태는 프로그램 실행 중에 변할 수 있는 변수의 값이나 메모리의 내용을 의미합니다. 상태를 관리하는 방법은 프로그램의 동작에 중요한 영향을 미칩니다.

2. **저장소(Store)란 무엇인가요?**

   - **답변:** 저장소는 변수의 값을 저장하는 메모리 공간을 의미합니다. 저장소는 변수의 위치(Loc)와 값(Val)을 매핑하여 관리합니다.

3. **저장소 전달 명세(Store-Passing Specification)란 무엇인가요?**

   - **답변:** 저장소 전달 명세는 저장소를 전달하면서 표현식을 평가하는 방법을 설명합니다. 예를 들어, `(value-of (const-exp n) ρ σ₀) = (n, σ₀)`는 상수 표현식을 평가할 때 저장소를 변경하지 않음을 나타냅니다.

4. **저장소 조작 연산(Store-Manipulating Operations)이란 무엇인가요?**

   - **답변:** 저장소 조작 연산은 저장소를 생성, 해제, 설정하는 연산을 의미합니다. 예를 들어, `newref`는 새로운 참조를 생성하고, `deref`는 참조된 값을 반환하며, `setref`는 참조된 값을 설정합니다.

5. **명시적 참조와 암시적 참조의 차이점은 무엇인가요?**

   - **답변:** 명시적 참조는 참조를 명시적으로 생성하고 조작하는 방식이며, 암시적 참조는 변수 자체가 참조로 동작하는 방식입니다. 예를 들어, `let counter = newref(0)`는 명시적 참조이고, `let counter = 0`는 암시적 참조입니다.

6. **L-Value와 R-Value란 무엇인가요?**

   - **답변:** L-Value는 변수의 위치를 나타내고, R-Value는 변수의 값을 나타냅니다. 예를 들어, `counter = counter + 1`에서 `counter`는 L-Value이고, `counter + 1`은 R-Value입니다.

7. **절차 적용 명세란 무엇인가요?**

   - **답변:** 절차 적용 명세는 절차를 호출할 때 변수의 값을 어떻게 전달하고 평가하는지 설명합니다. 저장소를 사용하는 경우, 절차 호출 시 저장소를 함께 전달하여 상태를 관리합니다.

8. **저장소 전달 명세의 예를 들어보세요.**

   - **답변:** 예를 들어, `(value-of (diff-exp e1 e2) ρ σ₀) = (v1 - v2, σ₂)`는 두 표현식 `e1`과 `e2`를 평가하여 그 차이를 계산하고, 저장소를 전달하는 과정을 설명합니다.

9. **저장소 조작 연산의 예를 들어보세요.**

   - **답변:** 예를 들어, `(value-of (newref-exp e) ρ σ₀) = (l, [l=v]σ₁)`는 새로운 참조를 생성하고, 해당 참조에 값을 저장하는 과정을 설명합니다.

10. **명시적 참조의 예를 들어보세요.**

    - **답변:** 예를 들어,

      ```scheme
      let counter = newref(0)
      	in let f = proc (x) setref(counter, deref(counter) + 1)
        	in let a = (f (f 1))
        		in deref(counter)
      ```

### 질문과 답변

1. **값에 의한 호출(Call-by-Value)이란 무엇인가요?**
   - **답변:** 값에 의한 호출은 함수 호출 시 매개변수의 값을 복사하여 전달하는 방식입니다. 함수 내부에서 매개변수의 값을 변경해도 원래 변수에는 영향을 미치지 않습니다.
2. **참조에 의한 호출(Call-by-Reference)이란 무엇인가요?**
   - **답변:** 참조에 의한 호출은 함수 호출 시 매개변수의 참조를 전달하는 방식입니다. 함수 내부에서 매개변수의 값을 변경하면 원래 변수의 값도 변경됩니다.
3. **이름에 의한 호출(Call-by-Name)이란 무엇인가요?**
   - **답변:** 이름에 의한 호출은 함수 호출 시 매개변수를 평가하지 않고, 함수 내부에서 매개변수가 사용될 때마다 표현식을 평가하는 방식입니다.
4. **지연 평가(Lazy Evaluation)란 무엇인가요?**
   - **답변:** 지연 평가는 표현식을 필요할 때까지 평가하지 않는 방식입니다. 이를 통해 불필요한 계산을 피하고, 성능을 향상시킬 수 있습니다.
5. **Thunk란 무엇인가요?**
   - **답변:** Thunk는 표현식과 환경을 캡슐화한 데이터 구조로, 지연 평가를 구현하는 데 사용됩니다. 필요할 때 Thunk를 평가하여 값을 얻습니다.
6. **명시적 참조와 암시적 참조의 차이점은 무엇인가요?**
   - **답변:** 명시적 참조는 참조를 명시적으로 생성하고 조작하는 방식이며, 암시적 참조는 변수 자체가 참조로 동작하는 방식입니다. 예를 들어, `let counter = newref(0)`는 명시적 참조이고, `let counter = 0`는 암시적 참조입니다.
7. **저장소 없이 절차 호출을 평가하는 방법은 무엇인가요?**
   - **답변:** 저장소 없이 절차 호출을 평가하려면 매개변수의 값을 복사하여 전달하고, 함수 내부에서 이를 사용하여 표현식을 평가합니다.
8. **참조에 의한 호출의 예를 들어보세요.**
   - **답변:** 예를 들어, `let p = proc (x) set x = 4 in let a = 3 in begin (p a); a end`에서 `p`는 `a`의 참조를 전달받아 `a`의 값을 `4`로 변경합니다.
9. **이름에 의한 호출의 예를 들어보세요.**
   - **답변:** 예를 들어, `letrec f = proc (x) (f (+ x 1)) in let g = proc (y) 10 in (g (f 0))`에서 `f`는 매개변수를 평가하지 않고, `g`가 호출될 때마다 `f`를 평가합니다.
10. **지연 평가의 예를 들어보세요.**
    - **답변:** 예를 들어, `letrec f = proc (x) (f (+ x 1)) in let g = proc (y) 10 in (g (f 0))`에서 `f`는 필요할 때만 평가되어 무한 루프를 피할 수 있습니다.

1. **Continuation Passing Style (CPS)이란 무엇인가요?**

   - **답변:** CPS는 함수 호출 시 현재 계산의 나머지 부분을 명시적으로 전달하는 스타일입니다. 함수는 결과를 반환하는 대신, 결과와 함께 나머지 계산을 수행할 Continuation을 호출합니다.

2. **일반적인 함수 정의를 CPS로 변환하는 방법은 무엇인가요?**

   - **답변:** 일반적인 함수 정의를 CPS로 변환하려면 함수의 각 호출에 대해 Continuation을 추가하고, 결과를 반환하는 대신 Continuation을 호출합니다. 예를 들어, 팩토리얼 함수 `fact`를 `fact/k`로 변환할 수 있습니다.

3. **CPS로 변환된 함수의 실행 과정을 설명해 주세요.**

   - **답변:** 예를 들어, `fact 3`의 실행 과정은 다음과 같습니다:

   1. `(fact/k 3 (end-cont))`
   2. `(fact/k 2 (fact1-cont 3 (end-cont)))`
   3. `(fact/k 1 (fact1-cont 2 (fact1-cont 3 (end-cont))))`
   4. `(fact/k 0 (fact1-cont 1 (fact1-cont 2 (fact1-cont 3 (end-cont)))))`

   8. `(apply-cont (end-cont) (* 3 (* 2 (* 1 1))))`
   9. 결과: `6`

4. **Continuation을 구현하는 방법은 무엇인가요?**

   - **답변:** Continuation을 구현하는 방법에는 데이터 타입과 패턴 매칭을 사용하는 방법과 절차적 표현을 사용하는 방법이 있습니다. 예를 들어, `apply-cont` 함수를 정의하여 Continuation을 적용할 수 있습니다.

   - **답변:** Continuation을 인라인화하려면 Continuation 함수를 직접 함수 본문에 포함시켜 호출을 단순화합니다. 예를 들어, `fact/k` 함수를 인라인화하여 `fact1-cont`를 직접 포함시킬 수 있습니다.

   1. `1 → f1 → f2 → f3 → end-cont → 6`

   ```scheme

      (lambda (n)
        (fact/k n (end-cont))))
    (define fact/k

### 질문과 답변

1. **타입 오류(Type Error)란 무엇인가요?**
   - **답변:** 타입 오류는 서로 다른 타입의 값을 연산하려고 할 때 발생하는 오류입니다. 예를 들어, `1 + "2"`는 정수와 문자열을 더하려고 시도하기 때문에 타입 오류가 발생합니다.
2. **타입 검사를 구현하는 방법은 무엇인가요?**
   - **답변:** 타입 검사는 각 표현식의 타입을 확인하고, 타입 규칙에 따라 올바른 타입인지 검사합니다. 예를 들어, `add-exp`의 타입 검사는 두 피연산자가 모두 정수 타입인지 확인합니다.
3. **타입 규칙이란 무엇인가요?**
   - **답변:** 타입 규칙은 각 표현식의 타입을 정의하는 규칙입니다. 예를 들어, `add-exp`의 타입 규칙은 두 피연산자가 모두 정수 타입일 때 결과도 정수 타입임을 나타냅니다.
4. **타입 검사와 해석의 차이점은 무엇인가요?**
   - **답변:** 타입 검사는 프로그램을 추상 값으로 실행하여 모든 가능성을 고려할 수 있으며, 타입 오류를 사전에 발견할 수 있습니다. 해석은 실제 값을 사용하여 프로그램을 실행합니다.
5. **타입 안전성이란 무엇인가요?**
   - **답변:** 타입 안전성은 프로그램이 실행 중에 타입 오류를 발생시키지 않음을 의미합니다. 타입 검사가 sound(타당)하면 프로그램이 타입 안전함을 보장할 수 있습니다.
6. **타당성과 완전성이란 무엇인가요?**
   - **답변:** 타당성은 타입 검사가 올바른 타입을 가진 모든 프로그램을 허용함을 의미합니다. 완전성은 올바른 타입을 가진 모든 프로그램을 타입 검사가 허용함을 의미합니다.
7. **정적 타이핑과 동적 타이핑의 차이점은 무엇인가요?**
   - **답변:** 정적 타이핑은 컴파일 시 타입 검사를 수행하여 타입 오류를 사전에 발견합니다. 동적 타이핑은 실행 시 타입 검사를 수행하여 실행 중에 타입 오류를 발견합니다.
8. **타입 검사가 타입 오류를 보고하지 않으면 프로그램이 타입 안전한가요?**
   - **답변:** 타입 검사가 sound(타당)하면 타입 오류를 보고하지 않은 프로그램은 타입 안전합니다. 즉, 실행 중에 타입 오류가 발생하지 않습니다.
9. **타입 검사의 타당성과 완전성을 모두 만족할 수 있나요?**
   - **답변:** 타입 검사의 타당성과 완전성을 모두 만족하는 것은 불가능합니다. 따라서 타입 검사는 타당하지만 완전하지 않거나, 완전하지만 타당하지 않을 수 있습니다.
10. **타입 검사의 예를 들어보세요.**
    - **답변:** 예를 들어, `proc (x) (+ x 1)`의 타입은 `(int → int)`입니다. 이는 `x`가 정수일 때 결과도 정수임을 나타냅니다.







# 15 type-checking-inf

1. **타입 검사(Type Checking)란 무엇인가요?**

   - **답변:** 타입 검사는 프로그램의 각 표현식이 올바른 타입을 가지는지 확인하는 과정입니다. 이를 통해 타입 오류를 사전에 발견할 수 있습니다.

2. **타입 추론(Type Inference)이란 무엇인가요?**

   - **답변:** 타입 추론은 명시적으로 타입을 지정하지 않은 경우에도 표현식의 타입을 자동으로 유추하는 과정입니다. 이를 통해 코드의 가독성을 높이고, 타입 안전성을 유지할 수 있습니다.

3. **타입 검사의 구현 예를 들어보세요.**

   - **답변:** 예를 들어, `add-exp`의 타입 검사는 두 피연산자가 모두 정수 타입인지 확인합니다.

   ```scheme
    (define type-of (lambda (exp tenv)
      (cases expression exp
        (add-exp (exp1 exp2)
          (let ((ty1 (type-of exp1 tenv))
                (ty2 (type-of exp2 tenv)))
            (check-equal-type! ty1 (int-type))
            (check-equal-type! ty2 (int-type))
            (int-type)))
        ...)))
   ```

4. **타입 추론의 예를 들어보세요.**

   - **답변:**

     ```scheme
     proc (f) proc (x) -((f 3), (f x))
     ```


   ```scheme
    t_0 = t_f → t_1
    t_1 = t_x → t_2
    t_3 = int, where t_3 is the type of (f 3)
    t_4 = int, where t_4 is the type of (f x)
    t_2 = int
    t_f = int → t_3
    t_f = t_x → t_4
   ```

   - **답변:**  타입 제약 해결 과정은 다음과 같습니다.

     2. t_1 = int → int
     3. t_3 = int
     4. t_4 = int

6. **타입 검사의 장점은 무엇인가요?**

   - **답변:** 타입 검사는 프로그램의 타입 오류를 사전에 발견하여 안정성을 높이고, 코드의 가독성을 향상시킵니다. 또한, 타입 시스템을 통해 프로그램의 의미를 명확히 할 수 있습니다.

   - **답변:** 타입 추론은 명시적으로 타입을 지정하지 않아도 자동으로 타입을 유추하여 코드의 가독성을 높이고, 타입 안전성을 유지할 수 있습니다. 이를 통해 개발자의 부담을 줄이고, 코드 작성 속도를 높일 수 있습니다.
